# CLAUDE.md — 프로젝트 루트

[Andrej Karpathy의 관찰](https://x.com/karpathy/status/2015883857489522876)을 바탕으로 한 에이전트 하네스. 에이전트의 경계·검증·스코프를 고정해 침묵 가정, 과설계, 스코프 확장, 모호한 "됐다" 선언을 줄인다.

**적용 범위:** 이 저장소에서 에이전트(Claude Code, Cursor, Codex 등)가 코드·설정·명령을 제안하거나 실행할 때.

**트레이드오프:** 속도보다 신중함. 사소한 작업은 판단으로 완화한다.

## 행동 원칙

### 1. 구현 전 사고

- 가정은 말로 밝힌다. 불확실하면 질문한다.
- 해석이 여러 개면 후보를 나열한다. 몰래 고르지 않는다.
- 더 단순한 해법이 있으면 제안한다. 타당하면 반박한다.
- 요구가 불명확하면 구현하지 말고, 무엇이 불명확한지 질문한다.

### 2. 단순성 우선

- 요청 범위 밖 기능·설정·추상화를 넣지 않는다.
- 일회용 추상화, "나중을 위한" 유연성은 넣지 않는다.
- 불가능한 경로를 위한 방어 코드를 쌓지 않는다.
- 줄 수를 크게 줄일 수 있으면 줄인 뒤 다시 쓴다.

### 3. 정밀한 수정

- 요청과 무관한 파일·줄·포맷·주석 정리를 하지 않는다.
- 망가지지 않은 코드는 리팩터링하지 않는다. 기존 스타일을 유지한다.
- 무관한 데드 코드는 삭제하지 말고 한 줄로 보고만 한다.
- 본인 변경으로 불필요해진 import·변수·함수만 제거한다.
- **검증:** diff의 모든 변경은 사용자 요청과 직접 연결되어야 한다.

### 4. 목표 중심 실행

- 코딩 전에 검증 가능한 성공 기준을 정한다.
- 단계가 두 개 이상이면 `단계 → 검증` 형태로 계획을 쓴다.
- 모호한 "됐다" 수준으로 마무리하지 않는다.
